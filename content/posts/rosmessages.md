## Custom Message Definitions

The iGYM system defines custom ROS2 messages in the `ap_interfaces` package for communication between the CV/hardware nodes and Unity.

### ap_interfaces/Pos
Used by the `player_detection` node to publish player positions on the `pos_raw` topic.

| Field | Type | Description |
|-------|------|-------------|
| `total` | int8 | Number of detected players |
| `timestamp` | uint64 | Capture timestamp in milliseconds |
| `x` | float64[32] | X positions of up to 32 players |
| `y` | float64[32] | Y positions |
| `player_id` | int8[32] | Unique player tracking IDs |
| `tag_id` | string[32] | RFID tag identifiers |
| `size` | float64[32] | Player blob radius/size |

### ap_interfaces/Button
Used by the `button` node to publish kick button states on the `kick_size` topic.

| Field | Type | Description |
|-------|------|-------------|
| `kick` | bool[32] | Button pressed state for up to 32 players |

### ap_interfaces/Score
Score data for game state synchronization.

| Field | Type | Description |
|-------|------|-------------|
| `player_score` | int16[32] | Individual player scores |
| `game_score` | int16[32] | Game-level scores |

### ap_interfaces/AddThreeInts (Service)
Service for passing game parameters between Unity and ROS2.

**Request and Response fields:**

| Field | Type | Description |
|-------|------|-------------|
| `ballspeed` | float32 | Ball speed parameter |
| `ballradius` | float32 | Ball radius |
| `goalsize` | float32 | Goal size |
| `friction` | float32 | Friction coefficient |
| `buttonspeed` | float32 | Button expansion speed |
| `capture` | float32 | Capture flag |
| `startstop` | float32 | Start/stop flag |

### Demo Messages (unity_robotics_demo_msgs)

These are used for testing the ROS2-Unity bridge integration:

**UnityColor** — Random RGB color values (`int32 r, g, b, a`)

**PosRot** — Position and rotation (`float32 pos_x, pos_y, pos_z, rot_x, rot_y, rot_z, rot_w`)

**PositionService** — Service returning random positions (Request: PosRot, Response: PosRot)

**ObjectPoseService** — Service returning object pose by name (Request: `string object_name`, Response: `geometry_msgs/Pose`)

### Note on Message Generation

The message files in the Unity project's RosMessages folder are auto-generated by Unity-ROS MessageGeneration from these `.msg` and `.srv` definitions. They should not be modified directly — edit the source definitions in `ap_interfaces/msg/` and `ap_interfaces/srv/` instead, then regenerate.
